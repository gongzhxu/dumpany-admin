# 系统配置子页面 — 服务配置管理

## 背景

当前 163 SMTP、腾讯云 SMS 等外部服务的配置项散落在不同位置：

| 服务 | 当前配置方式 | 问题 |
|---|---|---|
| 163 SMTP | system_config（smtp_host/smtp_port/smtp_user/smtp_pass） | 无管理页面，需直连 DB |
| 腾讯云 SMS | config.yaml（secret_id/secret_key/sdk_app_id/sign/template_id） | 改配置需重启服务 |
| SendGrid/Resend | config.yaml | 同上 |
| 支付宝 | 管理后台独立页面 | ✅ 有页面，但独立于系统配置 |

需要将这些服务配置统一放到 `system_config` 表中，并在管理后台增加一个**服务配置子页面**，统一管理所有第三方服务的凭据和参数。

## 页面位置

管理后台 → 系统配置 → 现有 system-config 页面中增加 Tab 或独立子页面（`/service-config`）

## 存储方式

所有配置以 JSON 格式存储在 `system_config` 表中：

| config_key | config_type | 说明 |
|---|---|---|
| `service_smtp` | json | 163 SMTP 配置 |
| `service_sms_tencent` | json | 腾讯云 SMS 配置 |
| `service_email_provider` | json | 邮件服务商配置（SendGrid/Resend） |

### 配置内容示例

```json
// service_smtp
{
  "host": "smtp.163.com",
  "port": 465,
  "user": "xxx@163.com",
  "pass": "xxxxxx"
}

// service_sms_tencent
{
  "secret_id": "xxx",
  "secret_key": "xxx",
  "sdk_app_id": "1400xxxxxx",
  "sign": "DumpAny",
  "template_id": "xxxxxx"
}

// service_email_provider
{
  "provider": "sendgrid",
  "sendgrid_key": "",
  "resend_key": "",
  "from": "noreply@dumpany.com"
}
```

## 管理后台页面

### 页面路由

`/admin/service-config`（或作为 `/admin/system-config` 的 Tab）

### 页面功能

- 左侧菜单：系统配置 → 服务配置
- 以卡片/分组形式展示每个服务
- 编辑按钮 → 弹出 Modal 编辑
- 保存后立即写入 system_config
- 支持新增/删除服务配置（后续扩展）
