# Swagger 账号管理页面

## 背景

Swagger 文档页面（`/swagger/index.html`）目前使用 BasicAuth 认证，账号密码存储在数据库 `settings` 表中（`swagger_user` / `swagger_pass`）。后端已提供 `PUT /api/v1/admin/settings/swagger` 接口，但 admin 前端缺少对应的管理页面。

## 需求

在管理后台左侧导航新增「系统设置」菜单项，包含 Swagger 账号密码修改功能。

## 功能描述

### 菜单
- 左侧导航新增 `系统设置` → `Swagger 账号` 菜单项
- 图标：`SettingOutlined`

### 页面

系统设置 → Swagger 账号：

```
┌─────────────────────────────────────┐
│  系统设置                            │
│  ┌─────────────────────────────────┐│
│  │ Swagger 文档访问账号             ││
│  │                                 ││
│  │ 用户名     [ 输入框  ]           ││
│  │ 密码       [ 输入框  ]           ││
│  │                                 ││
│  │  [ 保存 ]                       ││
│  │                                 ││
│  │ 提示：修改后下次访问 Swagger      ││
│  │ 文档时生效，当前登录不受影响。    ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

### 交互

- 页面加载时从 `GET /api/v1/admin/settings/swagger` 获取当前用户名（密码不回显）
- 用户名必填，最少 2 字符
- 密码可选，填写时才更新；最少 6 字符
- 点击保存调用 `PUT /api/v1/admin/settings/swagger`
- 成功提示，失败显示错误信息

### 后端接口（已有，不需改动）

```
PUT /api/v1/admin/settings/swagger
Authorization: Bearer <token>
Body: { "user": "admin", "pass": "newpassword" }
Response: { "code": 0, "msg": "success", "data": { "message": "Swagger credentials updated" } }
```
