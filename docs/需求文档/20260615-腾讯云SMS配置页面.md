# 腾讯云 SMS 配置管理页面

## 背景

腾讯云 SMS 配置当前只存在于 `config.yaml` 中，改配置需修改文件并重启服务。需要将 SMS 配置迁移到 `system_config` 表，并在管理后台提供独立的配置页面。

## 存储

存储在 `system_config` 表，`config_key = "service_sms_tencent"`，`config_type = "json"`。

```json
{
  "secretId": "xxx",
  "secretKey": "xxx",
  "sdkAppId": "1400xxxxxx",
  "sign": "DumpAny",
  "templateId": "xxxxxx"
}
```

## 页面路由

`/system-config/sms` — 放在左侧菜单"系统配置"下面作为子菜单项。

### 菜单结构

```
系统配置（父菜单，可展开）
├── 邮件配置  /system-config/smtp   ← 163 SMTP 配置
├── SMS配置   /system-config/sms    ← 腾讯云 SMS 配置
└── 通用配置  /system-config         ← 现有 key-value 表格
```

## 页面功能

参考支付宝配置页面（`/payment/alipay`）的风格：
- 卡片展示当前配置摘要（已配置/未配置状态）
- 编辑按钮 -> 弹出 Modal 填写/修改
- 保存后写入 `system_config`

### 编辑表单字段

| 字段 | JSON key | 类型 |
|---|---|---|
| SecretId | secretId | string |
| SecretKey | secretKey | string |
| SDK AppID | sdkAppId | string |
| 签名 | sign | string |
| 模板 ID | templateId | string |

### 状态判断

- `secretId` 和 `sdkAppId` 和 `sign` 和 `templateId` 都不为空 -> 已配置
- 否则 -> 未配置

## 后端

无需新增 handler，复用 `SystemConfigHandler` 的 List/Update 方法，前端用固定 key `service_sms_tencent` 读写。
