# 腾讯云服务配置管理页面

## 背景

当前腾讯云相关配置散落在多处：

| 服务 | 当前配置方式 | 位置 |
|---|---|---|
| SMS SecretId/SecretKey | config.yaml | sms.tencent |
| SMS 应用配置 | system_config | service_sms_tencent |
| COS SecretId/SecretKey | config.yaml | cos |
| COS Bucket/Region | config.yaml | cos |

SecretId/SecretKey 两组重复维护，且改配置需修改文件重启服务。需要统一管理。

## 存储结构

### COS 配置 JSON 结构

```json
{
  "bucket": "dumpany-feedback-xxxx",
  "region": "ap-guangzhou",
  "prefixes": [
    { "key": "feedback", "name": "反馈图片", "path": "feedback/" },
    { "key": "orders",   "name": "订单截图", "path": "orders/" }
  ]
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `bucket` | string | COS 存储桶 |
| `region` | string | 地域 |
| `prefixes[].key` | string | 程序标识，代码中通过此 key 引用（唯一） |
| `prefixes[].name` | string | 展示用名称，给人看的 |
| `prefixes[].path` | string | 实际 COS 路径 |

当前已有的 `feedback_prefix` 配置弃用，代码改为通过 key `"feedback"` 查找 path。

## 三种视图

每个配置卡片独立：

| 卡片 | 存储 key | 说明 |
|---|---|---|
| 凭证 | `tencent_credentials` | SecretId + SecretKey，SMS 和 COS 共用 |
| SMS | `tencent_sms` | SDK AppID / 签名 / 模板 ID |
| COS | `tencent_cos` | Bucket / Region / prefixes 列表 |

**无配置时** — 弹编辑 Modal，填完保存
**有配置时** — 摘要卡片 + 编辑按钮

COS 卡片的前缀列表支持在弹窗中编辑：

- 新增：填写 key / name / path，key 必填且不能重复
- 编辑：修改 name / path，key 不可修改
- 删除：Popconfirm 确认后删除

## 凭证复用

- COS 原本在 config.yaml 中有独立的 `cos.secret_id` / `cos.secret_key`
- 迁移后统一从 `tencent_credentials` 读取
- COS 卡片只配置 Bucket / Region / Prefixes
- 后端启动时从 system_config 读取凭证初始化 COS 客户端
