# 服务配置管理实现方案

## 存储设计

所有服务配置以 JSON 格式存入 `system_config` 表，与现有系统配置共用同一张表。

### 新增配置项

| config_key | config_type | 默认值 | 说明 |
|---|---|---|---|
| `service_smtp` | json | `{"host":"smtp.163.com","port":465,"user":"","pass":""}` | 163 SMTP 邮件服务 |
| `service_sms_tencent` | json | `{"secret_id":"","secret_key":"","sdk_app_id":"","sign":"DumpAny","template_id":""}` | 腾讯云短信服务 |
| `service_email_provider` | json | `{"provider":"resend","sendgrid_key":"","resend_key":"","from":"noreply@dumpany.com"}` | 邮件提供商 |

### 配置修改后实时生效

修改 `system_config` 后，`pkg/email/` 和 `pkg/sms/` 包在发送时从 `system_config` 读取最新配置，不需要重启服务。

## 后端 API

遵循路由规范，放在 `/admin/service-config` 下：

```
GET  /admin/service-config/list      → 返回所有服务配置
PUT  /admin/service-config/update    → 更新指定服务配置
```

### 复用现有结构

不需要新 handler/service/repo——直接复用 `system_config` 现有的一套：

| 现有 | 说明 |
|---|---|
| `handler/SystemConfigHandler` | List + Update，直接复用 |
| `service/SystemConfigService` | 已有，已处理缓存 |
| `repository/SystemConfigRepo` | 已有 |

只需要在 `routes.go` 添加一个路由别名，前端直接调 `/admin/service-config/list` 和 `/admin/service-config/update`，内部走同一套逻辑。

## 前端页面

### 页面路径

`/admin/service-config`

### 页面内容

与服务配置相关的 system_config 项（通过 `configKey` 前缀 `service_` 过滤）：

```
┌─────────────────────────────────────┐
│ 服务配置                              │
├─────────────────────────────────────┤
│                                     │
│ ┌─ 📧 163 SMTP 邮件 ──────────────┐ │
│ │ 状态: 已配置/未配置               │ │
│ │ 主机: smtp.163.com               │ │
│ │ 邮箱: xxx@163.com                │ │
│ │ [编辑]                           │ │
│ └──────────────────────────────────┘ │
│                                     │
│ ┌─ 📱 腾讯云短信 ─────────────────┐ │
│ │ 状态: 已配置/未配置               │ │
│ │ SDK AppID: 1400xxxxxx            │ │
│ │ 签名: DumpAny                    │ │
│ │ [编辑]                           │ │
│ └──────────────────────────────────┘ │
│                                     │
│ ┌─ ✉️ 邮件提供商 ─────────────────┐ │
│ │ 提供商: resend                   │ │
│ │ 发件邮箱: noreply@dumpany.com    │ │
│ │ [编辑]                           │ │
│ └──────────────────────────────────┘ │
└─────────────────────────────────────┘
```

### 编辑 Modal

点击 [编辑] → 弹出 Modal，根据服务类型渲染对应表单：

- 163 SMTP：host, port, user, pass
- 腾讯云 SMS：secret_id, secret_key, sdk_app_id, sign, template_id
- 邮件提供商：provider（下拉选择 sendgrid/resend）, sendgrid_key, resend_key, from

### 配置状态

- 必填字段全部不为空 → 显示"已配置 ✅"绿色标签
- 有必填字段为空 → 显示"未配置 ❌"红色标签
