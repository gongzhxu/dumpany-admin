# 支付宝配置编辑 Modal 灰色/黑色状态不更新问题

## 现象

编辑 Modal 打开后，所有字段应为灰色（表示未修改）。用户输入修改后，该字段应变为黑色。实际表现是字段始终为黑色，灰色从未生效，或者始终为灰色不变化。

## 根因

**Ant Design Form 的 `onValuesChange` 回调中，`form.getFieldsValue()` 返回的是 Form 内部 store 的引用，而 `setState` 的闭包捕获了过时的 `curr` 值，导致 re-render 后 `init[key] !== curr[key]` 永远是 false（灰色不转黑），或者初始就没置对。**

具体来说经历了三次尝试：

### 尝试一：`isDirty` 函数 + `form.getFieldsValue()`

```typescript
const isDirty = (initial, form, name) => {
  if (!initial) return true;
  const current = form.getFieldsValue();
  return initial[name] !== current?.[name];
};
```

**问题：** 没有触发 re-render 的机制。用户输入后组件不重渲染，`isDirty` 不会重新执行，颜色不变。加 `tick` state 也没用，因为 `tick` 的闭包和 `form.getFieldsValue()` 返回的引用不稳定。

### 尝试二：`init` / `curr` state + `onValuesChange` 更新 `curr`

```typescript
const [init, setInit] = useState({});
const [curr, setCurr] = useState({});

onValuesChange={(changed) => {
  const next = { ...curr, ...changed };
  setCurr(next);
}}
```

**问题：** `onValuesChange` 回调闭包捕获了过时的 `curr` 引用（创建回调时 `curr` 是空对象或旧值），导致 `...curr` 扩展的是旧数据，新输入的字段丢失。虽然颜色判断对了，但 `curr` 始终不对。

### 解决方案（最终）

**`onValuesChange` 中用 `form.getFieldsValue()` 实时读取 Form store，不使用闭包变量：**

```typescript
onValuesChange={() => {
  setCurr({ ...form.getFieldsValue() });
}}
```

同时 `console.log` 用 `_changed: any` 前缀避免 TS6133 警告。

## 总结

| 方案 | 问题 |
|---|---|
| `form.getFieldsValue()` 在 render 时直接调用 | 不触发 re-render |
| `tick` state | 闭包过时 |
| `onValuesChange` 里用 `...curr` | 闭包中 curr 过时 |
| ✅ `onValuesChange` 里用 `form.getFieldsValue()` | 实时读取 Form 内部 store，无闭包问题 |
